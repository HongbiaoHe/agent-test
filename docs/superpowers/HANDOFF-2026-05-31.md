## 一、本次做了什么（在 walking skeleton 之上）

### 1. `/agent` 三栏对话 UI（照抄 `demo/template` 风格）+ 多轮追加
- 三栏：会话侧栏 + 对话区 + 工具详情面板（manus 暖中性、响应式、ThemeToggle）。组件在 `apps/frontend/src/app/agent/_components/`、hooks 在 `_hooks/`、reducer 在 `_lib/thread.ts`。
- **统一 reducer**：把「`GET /conversations/:id` 历史」+「socket 实时事件」折叠成有序 thread items（user/assistant/tool/plan/error）。`_hooks/use-conversation.ts`。
- 后端新增追加入口 `POST /conversations/:id/messages`（租户校验 + 防并发 + 落 user 消息 + 入队续跑）。
- **多轮上下文关键坑**：deepagents 跑完一轮后**不在持久化 state 保留对话消息**（`agent.getState().values.messages` 实测为 0），同 `thread_id` 续跑拿不到上文。解决：**worker 每轮从 DB 重放完整对话历史**（`worker/agent.processor.ts` 的 `loadHistory()`，只回放 `type='message'` 的 user/assistant 文本）。`create()` 也把 goal 落成 seq 0 的 user 消息以统一。

### 2. `/agent/[id]` 动态路由（URL 驱动会话）
- 共享外壳 `_components/agent-shell.tsx`，由两个瘦页面驱动：`page.tsx`（新会话 id=null）/ `[id]/page.tsx`（具体会话，`useParams`）。
- 切换会话走 `router.push("/agent/[id]")`；`[id1]→[id2]` 同路由段不重挂载；**刷新按 URL 恢复原会话**。新会话首条消息发出后跳 `/agent/[新id]`。
- 主题用 `<html>.dark` 类初始化（`_hooks/use-theme.ts`），切路由不丢、SSR 无水合告警。

### 3. Passkey 登录（WebAuthn，后端实现）+ 邮箱兜底
- 后端 `@simplewebauthn/server`：`POST /auth/passkey/{register,login}/{options,verify}`（`auth/passkey.service.ts` + `passkey.controller.ts` + `dto/passkey.dto.ts`）。凭据存新表 `Authenticator`（关联 User），挑战临时存 Redis（flowId / userId，TTL 5min），验证通过由 `AuthService.signToken` 签发同一套后端 JWT；next-auth 的 Credentials 新增 `token` 路径把它存进 session（`frontend/src/auth.ts`）。
- 登录 discoverable（无需先填邮箱）；注册 `authenticatorAttachment: "platform"`（直接 Touch ID）。
- **RP 自适应**：前端按 `window.location.{hostname,origin}` 带 `rpId/origin`（`lib/api.ts`），后端用之（env 仅兜底）→ localhost 与隧道域名都自动正确。
- 保留**邮箱登录**作兜底（开发态，自动建号）。登录页 `app/login/page.tsx`（用户自行调过版式）。

### 4. `/command` → deepagents 原生 Skills 系统（**重点，按用户要求必须用原生**）
- 技能 = `apps/backend/skills/<name>/SKILL.md`（frontmatter `name`/`description` + 正文）。已建 `text-translate` / `text-summarize` / `email-compose`。
- `CommandRegistryService`（`src/commands/`）启动扫描目录建**只读**注册表；`GET /commands` 供前端 `/` 自动补全（按 `<domain>-<action>` 分组）；未知 `/command` 报 `COMMAND_NOT_FOUND`。
- **原生 SkillsMiddleware**：`createDeepAgent({ skills: ["/skills/"] })`（`agent/agent.factory.ts`）。worker 每轮把**所有技能 SKILL.md 经 invoke 的 `files` 注入 per-thread StateBackend 的 `/skills/` 下** → 中间件列出技能、agent 用 `read_file` **按需加载**（progressive disclosure，真实 tool 调用、会存 DB / UI 显示）。
- `/command` 显式触发：worker 把命令消息转成「请使用「X」技能完成（先 read_file 读 /skills/X/SKILL.md）：args」→ 稳定触发。展示用的原始 `/command` 仍存 DB。
- 前端 `/` 自动补全在 `_components/chat-thread.tsx`（输入 `/` 弹分组面板，仅非繁忙时显示）。
- **多租户隔离**：`files` 在每个会话各自的 StateBackend、按 `thread_id` 隔离；技能定义全局只读、注入时各自副本 → 不同用户互不影响。

### 5. LangSmith 追踪 + 命名

## 二、当前架构关键点（接手必读）

- **技术栈**：pnpm monorepo · `apps/backend`=NestJS 11 + Prisma + MySQL + Redis(stack) + BullMQ + Socket.IO + deepagents 1.10.2 + Gemini · `apps/frontend`=Next 16 + React 19 + Tailwind4 + shadcn + TanStack Query + next-auth v5(beta) + @simplewebauthn。
- **数据模型**（`prisma/schema.prisma`）：Tenant / User / **Authenticator**(passkey) / Conversation / Message / Approval。
- **多轮**：deepagents 不跨 run 留消息 → worker 每轮从 DB 重放历史（lossy：只回放文本消息，工具调用不回放；技能靠每轮重新注入 + 重新 read_file）。
- **Skills**：原生 SkillsMiddleware + 每轮 `files` 注入 + read_file 加载。**不要改回手动往 system prompt 拼正文**（用户明确否决过）。
- **鉴权**：后端签 JWT；passkey/邮箱都经 next-auth Credentials 桥进 session cookie；REST 带 `Authorization: Bearer`、socket 带 `auth.token`；按 `tenantId` 隔离。

## 三、环境与运行（务必照做）

- **Node 必须 v22**：每条 shell 前 `export PATH="$HOME/.nvm/versions/node/v22.21.1/bin:$PATH"`（默认 shell 是 v14）。
- **起服务**用 `.claude/launch.json` 的 `preview_start`：`backend`、`frontend`(dev)、`frontend-prod`(`next start`，隧道演示用)。改后端代码 watch 会重编；**改 `.env`/新增模块务必 `preview_stop`+`preview_start` 重启**。
- **docker**：`docker compose -f docker-compose.dev.yml up -d`（mysql + redis-stack）。
- **后端 `.env`**（gitignore）：`GOOGLE_API_KEY`、`GOOGLE_GENAI_MODEL`、`DATABASE_URL`、`REDIS_URL`、`CORS_ORIGIN`、`WEBAUTHN_RP_ID/RP_NAME/ORIGIN`（本机=localhost；客户端会自适应覆盖）、`LANGSMITH_*`、`SKILLS_DIR`(默认 `<cwd>/skills`)。
- **前端 `.env.local`**（gitignore）：`NEXT_PUBLIC_API_BASE_URL=/api-backend`、`BACKEND_INTERNAL_URL=http://localhost:3001`、`AUTH_SECRET`、`AUTH_TRUST_HOST=true`、`AUTH_URL`（隧道域名；本机+隧道都兼容因登录用 redirect:false）。
- **localhost ↔ 隧道切换**：WebAuthn 自适应无需改；切隧道做 dev 时把 `next.config.ts` 的 `allowedDevOrigins` 设隧道域名（仅 dev 需要），或用 `frontend-prod`。
- **生产模式注意**：用 `frontend-prod` 时改前端代码要 `pnpm --filter frontend build` 再重启。
- Prisma 迁移：`cd apps/backend && npx prisma migrate dev`。
